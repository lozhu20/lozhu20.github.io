操作系统的圆角内卷

lozhu
2025-07-13
日志随笔
UI, 圆角

操作系统的圆角内卷，真的有必要吗？

很早之前，苹果首先使用圆角矩形，带来了很不一样的视觉效果和使用体验。直到最近两年之前，圆角矩形还比较保守，整个矩形只是将直角部分的很小一部分做成了圆润的弧线。使用了很多年的苹果手机，这套 UI 始终给人很舒服的感觉，既简约又大气。

大概两三年前，某一次更新 iOS 之后，发现通知栏的通知相比之前圆了很多，变得相当违和。本来整个通知就 3 行文字，高度非常小，配上一个超级大的圆角，真是丑出天际。每次下拉通知栏都要提前硬着头皮，捂着鼻子，转眼好几年过去了，也没有完全适应。

到如今，macOS 、Windows、某些 Linux 发行版、某些手机厂家的安卓等各种操作系统对这违和感满满的大圆角趋之若鹜，大圆角似乎成了现代操作系统设计的万能钥匙。我不禁要问：各种现代操作系统对大圆角的执着追求，真的有必要吗？

最近我的 mini 主机每次开机都不定期弹出提醒，让我尽快升级到 Win11。21 年就发布的 Win11，我甚至都还没在虚拟机里装了体验一下。去年公司发了一个笔记本，里面装的是 Win11，才有机会体验一下这个新操作系统。相比 Win10，我都不知道微软是不是在搞反向升级，实用、美观、特色荡然无存。Win11 上窗口、图标乃至任务栏都使用了大圆角。有些人解释说大圆角匹配了移动设备的屏幕边框，带来了更柔和、更友好的视觉体验，消解了传统直角界面的冰冷和僵硬，让 Windows 系统终于跟上了时代审美。但我只觉得这不过是盲目跟风，丢掉了自身硬朗、高效的风格。所有窗口都被圆润化，Windows 棱角分明、简洁直接的操作逻辑也被一并磨平。大圆角没让我的注意力更集中于元素的内容，而是将我的注意力吸引到了这个丑陋的大圆角上。

以前总是将 Win10 和 macOS 对比，只看到 Win10 的缺点，使用了一段时间的 Win11 之后，才开始体会到原来 Win10 是这么好用，Win 主机的开机频率都变高了许多。要是 Win12 依然是 Win11 这套 UI，那真是不得不考虑转 Linux 了。

各个厂家都想在手机、平板、电脑上搞 UI 的大一统。可是每个平台有每个平台的使用场景，无脑强行统一只会降低体验。不同厂家抄来抄去也是个大问题。某个大厂搞出个什么新奇的点子，不论是好是坏，其他厂商迫不及待的立马抄过来，唯恐自己赶不上潮流。什么刘海屏、灵动岛、买设备不送充电头，各种毫无下限。这种你追我赶的抄来抄去，各个操作系统能保持自己的特色就见鬼了。

真别无脑上大圆角了，做点实用又有自己风格的 UI 吧。圆角内卷，真没必要。
